# (Diary)

这是一个“零维护”的静态日记网站。
**只需往 GitHub 文件夹里丢 `.md` 文件，网页就会自动显示，不需要运行任何脚本。**

## 🚀 核心原理

前端页面 (`assets/js/app.js`) 会在每次打开时，自动调用 **GitHub API**，扫描 `posts/` 文件夹下的所有文件，并动态生成时间轴。

## 🛠 配置（仅需一次）

打开 `assets/js/app.js`，修改顶部的配置区域：

```javascript
const CONFIG = {
    repoOwner: "你的GitHub用户名",  // 例如: "zhangsan"
    repoName: "你的仓库名",        // 例如: "diary-site"
    branch: "main",               // 通常是 main
    basePath: "posts"
};
```

## ✍️ 如何写日记

**极其简单：**

1.  打开这个 GitHub 仓库。
2.  进入 `posts/` 文件夹。
3.  你可以随便创建文件夹（比如 `2024/01/`），或者直接把 `.md` 文件丢进去。
4.  前端会自动尝试解析路径里的日期（比如 `2024` 和 `01`），如果解析不出来，就会归类到“其他”。
5.  **支持拖拽上传**：直接在 GitHub 网页上点击 "Upload files"，把写好的 Markdown 文件拖进去即可。

## ⚠️ 注意事项

1.  **公开仓库 (Public Repo)**：推荐使用 Public 仓库。
    *   GitHub API 对公开仓库的读取是免费且方便的。
    *   如果你使用 Private 仓库，前端直接调用 API 会失败（报 404），除非你为了这个小网站去折腾复杂的 Token 认证（不推荐暴露 Token 在前端）。
    *   *隐私提示*：如果你不想让别人搜到，可以把仓库名起得复杂一点，只要不把网址发给别人，也没人会特意来翻你的日记。

2.  **API 限制**：
    *   GitHub API 限制每小时每 IP 请求 60 次。对于两个人写日记、看日记来说，完全够用了。
